A sequence of aggregate operations is known as a 
Pipeline
Each pipeline contains zero or more ___ operations 
Intermediate
Each pipeline ends with a ___ operation 
Terminal
What kind of operation produces another stream as its output? 
Intermediate
Describe one way in which forEach aggregate operation differs from the enhanced for statement and iterator 
The forEach aggregate operation lets the system decide "how" the iteration takes place. 
	Using aggregate operations lets you focus on "what" instead of "how."
True or False: A stream is similar to a collection in that it is a data structure that stores elements. False. Unlike a collection, a stream is not a data structure. It instead carries values from a source through a pipeline
Identify the intermediate and terminal operations in this code: double average = roster .stream().filter(p -> p.getGender() == Person.Sex.MALE).mapToInt(Person::getAge).average().getAsDouble(); Intermediate: filter, mapToInt, Terminal: average The terminal operation average returns an OptionalDouble. The getAsDouble method is then invoked on that returned object. It is always a good idea to consult the API Specification for information about whether an operation is intermediate or terminal
The code p -> p.getGender() == Person.Sex.MALE is an example of what? lambda expression
The code Person::getAge is an example of what method reference
Terminal operations that combine the contents of a stream and return one value are known as what Reduction operations
Name one important difference between the Stream.reduce method and the Stream.collect method Stream.reduce always creates a new value when it processes an element. Stream.collect modifies (or mutates) the existing value
If you wanted to process a stream of names, extract the male names, and store them in a new List, would Stream.reduce or Stream.collect be the most appropriate operation to use? 
The collect operation is most appropriate for collecting into a List
Example:
List<String> namesOfMaleMembersCollect = 
roster.stream().filter(p -> p.getGender() == Person.Sex.MALE).map(p -> p.getName()).collect(Collectors.toList());
True or False: Aggregate operations make it possible to implement parallelism with non-thread-safe collections True, provided that you do not modify (mutate) the underlying collection while you are operating on it
Streams are always serial unless otherwise specified. How do you request that a stream be processed in parallel? Obtain the parallel stream by invoking parallelStream() instead of stream()
What is finalize() method 
It is possible to define a method that will be called just before an object's final destruction by the 
garbage collector. This method is called finalize( ), 
and it can be used to ensure that an object terminates cleanly
